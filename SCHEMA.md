# Ontology Graph Studio — Data Model Reference

> Canonical reference for all PostgreSQL tables, Neo4j node/edge schemas, and domain vocabularies.
> Keep this file in sync with `backend/app/db/models.py` and `backend/app/extraction/base.py`.

---

## PostgreSQL Tables

Managed by SQLAlchemy 2.0 async ORM. Tables auto-created on startup via `metadata.create_all()`.
Connection string: `DATABASE_URL` in `.env` (default: `postgresql+asyncpg://...`).

---

### `documents`

Primary record for every ingested document.

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID PK | Generated by Python |
| `filename` | VARCHAR | Original filename |
| `source_type` | VARCHAR | `file`, `text`, `url` |
| `mime_type` | VARCHAR | Detected via python-magic |
| `raw_text` | TEXT | Full extracted text |
| `word_count` | INTEGER | Computed at ingest |
| `language` | VARCHAR | Detected (stub) — defaults to `en` |
| `metadata_json` | TEXT | Extra key-value pairs (JSON string) |
| `created_at` | TIMESTAMP | UTC |

---

### `chunk_embeddings`

OpenAI vector embeddings, one row per text chunk per document.

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID PK | |
| `document_id` | UUID FK → `documents.id` | Cascade delete |
| `chunk_index` | INTEGER | 0-based position in document |
| `text` | TEXT | Raw chunk text |
| `embedding` | Vector(1536) | pgvector column — text-embedding-3-small |
| `token_count` | INTEGER | Estimated token count |
| `metadata_json` | TEXT | e.g. `{"start_char": 0, "end_char": 512}` |

**Indexes**: `embedding` column uses pgvector cosine distance index (created at startup).

## Domain Vocabularies

Defined in `backend/app/extraction/base.py` as `frozenset` (immutable at runtime).

### Entity Types

```python
ENTITY_TYPES = frozenset({
    "Company",
    "Person",
    "Product",
    "Contract",
    "Location",
    "Technology",
    "Policy",
    "Regulation",
})
```

> To add a new entity type: update `ENTITY_TYPES` in `base.py` **and** update PROMPT-001 in this file and in `entity_extractor.py`.

### Knowledge Issue Types

Detected by `KnowledgeEvolutionEngine.detect_issues()`:

| Type | Description | Severity |
|------|-------------|----------|
| `low_confidence` | Entity or relationship with confidence < threshold | medium |
| `orphan_node` | Entity with no relationships | low |
| `missing_properties` | Entity missing expected attributes for its type | medium |
| `duplicate_names` | Multiple entities with same name and same type | high |
| `sparse_subgraph` | Entity with fewer than 2 connections | low |
| `conflicting_types` | Same name used with multiple entity types | high |

---

### Ontology Proposal Types

Generated by `KnowledgeEvolutionEngine.generate_proposals()`:

| Type | Parameters in `detail_json` | When generated |
|------|---------------------------|----------------|
| `add_class` | `name`, `description`, `parent_class` | New entity type pattern detected |
| `merge_class` | `class_a`, `class_b`, `merged_name` | Two classes with high overlap |
| `rename_class` | `old_name`, `new_name`, `reason` | Naming inconsistency detected |
| `add_relationship` | `relationship_type`, `source_class`, `target_class` | New relationship pattern detected |

## Neo4j Graph Schema

### Node Properties

All entity nodes share this base schema regardless of label:

| Property | Type | Source |
|----------|------|--------|
| `pg_id` | String (UUID) | `extracted_entities.id` |
| `name` | String | `extracted_entities.name` |
| `entity_type` | String | `extracted_entities.entity_type` |
| `confidence` | Float | `extracted_entities.confidence` |
| `document_id` | String (UUID) | `extracted_entities.document_id` |

**Node labels**: PascalCase version of `entity_type` — e.g. `Company`, `Person`, `Technology`.
Label conversion handled by `GraphBuilderService._safe_label()` in `backend/app/graph/writer.py`.

---

### Relationship Properties

All edges use the type `RELATES` with a `type` property (current model). This allows flexible querying but trades off query performance.

| Property | Type | Source |
|----------|------|--------|
| `type` | String | `extracted_relationships.relationship_type` |
| `confidence` | Float | `extracted_relationships.confidence` |
| `document_id` | String (UUID) | `extracted_relationships.document_id` |
| `pg_id` | String (UUID) | `extracted_relationships.id` |

> **Note (see ADR-005-typed-edges.md)**: A future migration should convert generic `RELATES` edges to typed relationships (`:WORKS_FOR`, `:USES`, etc.) for better Cypher query performance. Tracked in ROADMAP Step 5.

---

### Useful Cypher Queries

```cypher
-- All nodes
MATCH (n) RETURN n LIMIT 50

-- Entity type distribution
MATCH (n)
WHERE n.entity_type IS NOT NULL
RETURN n.entity_type, count(n) AS count
ORDER BY count DESC

-- All edges for a document
MATCH (a)-[r:RELATES]->(b)
WHERE r.document_id = '<document_id>'
RETURN a.name, r.type, b.name

-- N-hop neighborhood around a node
MATCH path = (n {pg_id: '<entity_id>'})-[*1..2]-(neighbor)
RETURN path

-- Low-confidence relationships
MATCH (a)-[r:RELATES]->(b)
WHERE r.confidence < 0.5
RETURN a.name, r.type, b.name, r.confidence
ORDER BY r.confidence ASC

-- Companies that use a specific technology
MATCH (c {entity_type: 'Company'})-[r:RELATES {type: 'USES'}]->(t {name: 'AWS'})
RETURN c.name, r.confidence

-- Orphan nodes (no relationships)
MATCH (n)
WHERE NOT (n)-[]-()
RETURN n.name, n.entity_type
```
