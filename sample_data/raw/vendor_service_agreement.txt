ARTICLE 1 — SCOPE OF SERVICES

1.1 Service Summary

CloudOps Solutions will provide Hartwell Manufacturing with managed cloud infrastructure, continuous integration/continuous delivery (CI/CD) pipeline management, and 24×7 site reliability engineering (SRE) services, as detailed below.

1.2 Managed Services

A. AWS Infrastructure Management
   - Management and optimization of Hartwell's Amazon Web Services (AWS) environment, including EC2, RDS, S3, VPC, IAM, and CloudFront
   - Reserved Instance and Savings Plans procurement and management
   - Multi-region architecture: us-east-1 (primary), us-west-2 (DR)
   - Monthly cost optimization reports using AWS Cost Explorer and CloudHealth by VMware

B. CI/CD Pipeline Engineering
   - Design, implement, and maintain CI/CD pipelines using GitHub Actions and AWS CodePipeline
   - Container orchestration via Amazon EKS (Kubernetes 1.29)
   - Artifact management with AWS ECR and JFrog Artifactory
   - Infrastructure as Code (IaC) using Terraform 1.7 and Ansible 2.16

C. Monitoring and Observability
   - Full-stack observability using Datadog APM, Infrastructure, and Log Management
   - Custom dashboards and SLO/SLI tracking
   - Incident alerting via PagerDuty with defined escalation policies
   - Synthetic monitoring via Datadog Synthetics for 14 critical business endpoints

D. Security and Compliance Posture Management
   - AWS Security Hub with CIS AWS Foundations Benchmark v2.0 compliance checks
   - Vulnerability scanning with Aqua Security Trivy on all container images
   - Secret management via HashiCorp Vault Enterprise 1.15
   - Penetration testing coordination (annual, third-party: Bishop Fox LLC)

E. Database Administration
   - Management of Amazon RDS for PostgreSQL 15 (production) and Aurora MySQL 3.0 (legacy ERP)
   - Automated backup and point-in-time recovery (PITR) with 35-day retention
   - Query performance analysis and index optimization
   - Migration support for SAP S/4HANA integration

ARTICLE 4 — POLICIES AND GOVERNANCE

4.1 Information Security Policy

All CloudOps personnel with access to Hartwell systems must comply with:
  a) Hartwell Information Security Policy HW-SEC-001 (rev. 4.2, effective January 2024)
  b) NIST SP 800-53 Rev. 5 control framework (moderate baseline) — Hartwell's adopted security standard
  c) CIS Controls v8 Implementation Group 2

4.2 Data Classification and Handling Policy

  - All data processed under this Agreement is classified as Confidential per Hartwell Data Classification Policy HW-DATA-003
  - Vendor shall not store Hartwell data outside AWS us-east-1 and us-west-2 regions without written consent
  - PII data (as defined under GDPR Article 4(1) and CCPA Section 1798.140) must be handled per Hartwell Privacy Policy HW-PRIV-002

4.3 Access Control Policy

  - Principle of least privilege applies to all access grants
  - All human access via SSO using Okta with MFA enforcement (Okta Verify or hardware FIDO2 key)
  - Privileged access via HashiCorp Vault dynamic secrets — no static long-lived credentials permitted
  - Access reviews conducted quarterly by Patricia Delgado and Ananya Krishnamurthy

4.4 Acceptable Use Policy

CloudOps personnel shall not use Hartwell computing resources for any purpose other than service delivery under this Agreement. Use of generative AI tools (including ChatGPT, GitHub Copilot, and similar) on Hartwell codebases requires approval under Hartwell AI Governance Policy HW-AI-001 (effective July 2024).

4.5 Incident Disclosure Policy

CloudOps shall notify Hartwell within 1 hour of confirming any security incident affecting Hartwell systems, and within 72 hours for any personal data breach, in compliance with GDPR Article 33 and applicable state breach notification laws (Michigan Identity Theft Protection Act, MCL 445.72; California Civil Code §1798.82).

4.6 Subcontractor Policy

CloudOps shall not engage subcontractors for services under this Agreement without prior written consent from Patricia Delgado. Approved subcontractors as of the Effective Date: none.
